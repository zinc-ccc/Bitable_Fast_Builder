# Bitable-Fast-Builder (v1.0.0)

## 项目目标
实现“自然语言描述 -> 规范化多维表字段 -> 自动创建”。

## 字段设计准则 (Design Principles)
本项目优先考虑**看板美观度与用户体验**。在满足系统调用的基础下，遵循以下准则：
- **可读性第一**：避免冗长的符号堆砌，优先使用直观的中文描述。
- **推荐规范**：
  - 如果需要区分层级，建议使用冒号 `:` (如 `入职日期:计划`)。
  - 避免在看板上显示复杂的括号 `()` 或 连字符 `-`，以防解析异常。
- **符号要求**：符号尽量保持半角英文，避免中文全角符号在代码处理时出现乱码。

### 示例
- 推荐：`计划入职日期` 或 `入职日期:计划`
- 不推荐：`入职日期(计划)-确认版`

## 常用标准表头
- `岗位唯一码`
- `关联组织_岗位类型`
- `关联_架构`

## 功能规划
1. **需求解析**：通过飞书机器人接收自然语言描述。
2. **字段生成**：根据描述自动设计符合规范的表头字段。
3. **流程设计**：规划自动化流程及看板视图。
4. **自动搭建**：利用飞书 API 自动创建多维表及配置。
